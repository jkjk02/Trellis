# 选择式交互规范

> 所有用户决策点必须使用选择式 UI (AskUserQuestion)，禁止要求用户手动输入

---

## 原则

1. **选择优先**: 任何需要用户做决定的场景，提供 2-4 个预设选项
2. **始终保留 Other**: 工具自动提供 "其他" 选项，用户可自由输入
3. **可并行提问**: 独立的决策点可以合并为一次多问题交互
4. **简短描述**: 每个选项附带一行说明，帮助用户快速理解

---

## 标准决策点清单

### Phase 1: Plan

| 决策点 | 推荐选项 |
|--------|----------|
| 任务分类 | 简单任务 / 复杂任务 / 纯讨论 |
| 是否创建 Trellis 任务 | 创建任务进入规划 / 跳过直接做 / 先讨论 |
| 需求确认 | 确认需求 / 需要补充 / 拆分任务 |
| 任务拆分方式 | 前后端分离 / 按功能模块 / 按优先级 |
| 技术方案选择 | 方案 A / 方案 B / 需要更多信息 |

### Phase 2: Execute (三模型协作)

| 决策点 | 推荐选项 |
|--------|----------|
| 任务分解确认 | 同意拆分 / 调整比例 / 全部给一个模型 |
| Worker 错误处理 | 重试 / 换模型 / Claude 直接做 / 放弃 |
| 实现结果审阅 | 进入测试 / 返工后端 / 返工前端 / 全部返工 |
| 测试结果处理 | 通过 / 修复后重测 / 放弃该功能 |
| 终审结果 | 提交 / 修改后再审 / 拆分提交 |

### Phase 3: Finish

| 决策点 | 推荐选项 |
|--------|----------|
| 提交方式 | 单次提交 / 分文件提交 / 前后端分别提交 |
| Spec 更新 | 更新现有 spec / 创建新 spec / 跳过 |
| 是否推送 | 推送到远程 / 仅本地提交 / 创建 PR |

## 实现方式

在 Claude Code 中使用 `AskUserQuestion` 工具：

```json
{
  "questions": [
    {
      "header": "简短标签",
      "question": "完整问题描述？",
      "multiSelect": false,
      "options": [
        {"label": "选项名 (推荐)", "description": "一行说明"},
        {"label": "选项名", "description": "一行说明"}
      ]
    }
  ]
}
```

### 规则

- `header` 最多 12 字符
- `options` 2-4 个选项
- 推荐选项放第一位，标签末尾加 `(推荐)`
- `multiSelect: true` 用于非互斥选择 (如"哪些文件需要审核？")
- 有代码/布局对比时使用 `preview` 字段

---

## 禁止的交互模式

| 禁止 | 替代 |
|------|------|
| "请输入你的选择" | 使用 AskUserQuestion 选项 |
| "你觉得怎么样？" | 提供具体选项让用户选 |
| 开放式问题无选项 | 至少提供 2 个建议选项 |
| 连续问 5+ 个问题 | 合并为 1-4 个并行问题 |
